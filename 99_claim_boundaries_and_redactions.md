# Claim Boundaries And Redactions

## Why this file exists

The public evidence package must be strong without becoming careless. These excerpts are designed to show real work while protecting private operational details and avoiding inflated claims.

## Redacted or not included

- API keys, credentials, secrets and account identifiers.
- Phone numbers and private messaging targets.
- Full local filesystem paths.
- Private Drive/iCloud folder names where they would identify personal storage.
- Full unpublished documents where a short excerpt is enough to prove the point.

## Claim boundaries

| Area | What is claimed | What is not claimed |
|---|---|---|
| Autowatcher v7.0 | Multi-agent notebook orchestration using sync infrastructure, Papermill and persistent task/result folders. | Not production SLA cloud infrastructure. |
| Miguel Core | Local AI operating architecture with memory, tools, retrieval and external control concepts. | Not a finished commercial assistant product. |
| WhatsApp/OpenClaw | Remote-control flow and quality-loop design. | Not public release of private phone/config details. |

## Reviewer interpretation

The intended signal is engineering judgment under constraints: build working systems, expose evidence, set boundaries, and avoid turning exploratory work into claims that the artifacts cannot support.
