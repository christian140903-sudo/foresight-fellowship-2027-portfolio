Quality loop

async def orchestrate(message):
    understanding = await understand(message)
    result = await execute(understanding.work_order)
    for _ in range(MAX_EVALUATION_LOOPS):
        evaluation = await evaluate(result, understanding.success_criteria)
        if evaluation.score >= PASS_THRESHOLD:
            break
        result = await execute(evaluation.next_work_order)
    return format_for_whatsapp(result, evaluation)

def complete(task):
    for tier in [LOCAL, FREE_API, PAID]:
        candidates = available_models(tier, task.strategy, quotas, cooldowns)

Security-relevant design pattern

The recurring design pattern is separation of authority. A human-facing message, voice command, or local UI event should not become direct arbitrary execution. It is first interpreted, normalized, routed, executed through scoped tools, evaluated, logged or summarized, and only then returned through the interface. The current work is a personal prototype ecosystem, but that pattern is exactly the kind of practical experience secure agent systems require.

What still needs hardening

  • Public demo mode with synthetic data and credentials disabled.
  • Explicit permission labels for every tool and route.
  • Confirmation thresholds for destructive or irreversible actions.
  • Machine-readable audit traces for model choice, tool calls, evaluator judgments, and memory provenance.
  • Benchmarks for router quality, voice latency, memory retrieval, and evaluator-loop failure modes.

Why sanitized excerpts are better than screenshots

Raw screenshots would be weak evidence because they show surface output without mechanism. The appendix instead uses sanitized source shapes, architecture tables, module responsibilities, and claim boundaries. That is more useful to a technical evaluator: it shows what the systems were meant to do, where the evidence sits, and which claims are intentionally withheld because they would require fresh tests or private material.
