Online stellen und teilen

Lade immer den kompletten Ordner `Fellowship-Bewerbung_FINAL` hoch, nicht einzelne HTML-Dateien. Die Website, beide Sprachen und alle PDFs liegen zusammen in diesem Ordner.

Am einfachsten: Netlify Drop

  1. app.netlify.com/drop öffnen.
  2. Den kompletten Ordner hineinziehen.
  3. Deploy-URL abwarten.
  4. Diesen Link in Bewerbung, Mail oder Profil einfuegen.

GitHub Pages

  1. Public Repository erstellen.
  2. Ordnerinhalte hochladen.
  3. Pages aus main/root aktivieren.
  4. github.io-Link verwenden.

Cloudflare Pages

  1. Pages-Projekt erstellen.
  2. Direct Upload wählen.
  3. Kompletten Ordner hochladen.
  4. Deployment-URL teilen.
Nicht hochladen: den internen Control-Ordner, rohe Quellordner, `.env`-Dateien, Tokens, private lokale Pfade oder alte Draft-Ordner.
Als evidenzorientiertes Portfolio vorbereitet. Öffentliche Dateien enthalten bewusst keine privaten Quellpfade, Zugangsdaten, privaten Kontaktdaten oder lokalen Links.
