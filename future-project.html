Future Project: ECSA Predictive AMR AI Architecture

A staged AI-for-science project direction: probabilistic modeling of antimicrobial resistance dynamics using graph/state-space models, uncertainty, stability, lab-in-the-loop validation, and regulatory/open-science governance.

Boundary

This is an architecture/proposal and IP-style drafting body of work. It should be developed into a proof of concept before any clinical or decision-support claims are made.

Core model

  • Nodes represent microbial resistance or sensitivity states.
  • Edges represent transitions, collateral sensitivity, cross-resistance, risk, or time-dependent weights.
  • Each state carries resistance R, uncertainty U/sigma, and stability S.
  • New measurement data updates node parameters, edge probabilities, and graph structure.
  • Outputs include resistance trajectories, risk indicators, intervention options, and visualizations.

Lab-in-the-loop cycle

Gold-data generation through standardized wet-lab experiments Bayesian inference updates probabilities and uncertainty intervals GNN layer detects clusters and predicts missing relationships Optional RL simulates stable intervention paths Hypotheses return to laboratory validation Governance layer tracks audit, bias, FAIR, MDR / AI Act readiness

AMR-HORIZONT as project execution evidence

AMR-HORIZONT is a separate FFG Impact Innovation Social Track package for a participatory One-Health decision framework. It shows 12-month project planning, budget logic, work packages, stakeholder strategy, LOI materials, evaluator Q&A, red-flag mitigation, concrete outputs, and scenario-based testing.

Use in the Fellowship package as secondary proof of project organization and governance thinking, not as the main Secure AI evidence.

Proof-of-concept roadmap

Phase 1: data and simulator

Build a synthetic AMR state-space simulator with explicit resistance, uncertainty, and stability parameters. Use public literature examples and synthetic transition matrices before touching sensitive clinical data.

Phase 2: model layer

Implement Bayesian updating and graph link prediction on the synthetic/public dataset. Measure calibration, uncertainty behavior, and failure cases.

Phase 3: governance layer

Add model cards, data provenance, audit logs, red-team tests, and decision-support disclaimers before any real-world stakeholder presentation.

Risk controls

RiskControl
Overclaiming medical valueKeep all outputs as research/prototype until externally validated.
Data bias or sparse gold dataUse uncertainty estimates, stratified sampling, and external validation.
Runaway feedback between model and lab hypothesesHuman-in-the-loop review, re-baselines, and drift detection.
Privacy/GDPR exposureBegin with public/synthetic data and design pseudonymization/provenance before clinical data.
Regulatory mismatchDocument intended-use boundaries and SaMD/AI Act implications from the start.

Why ECSA belongs in the Fellowship package

ECSA should not lead the Fellowship application, because the Secure AI proof is stronger in the agent-infrastructure work. It belongs as the future project answer: a concrete AI-for-science direction where probabilistic modeling, uncertainty, lab feedback, governance, and model accountability all matter.

The proposed system models antimicrobial resistance as a dynamic graph/state-space problem. Instead of returning one black-box prediction, it carries resistance, uncertainty, and stability through the model, updates those parameters with new evidence, and returns risk/trajectory information together with the limits of the current model.
