Deploy and Share Instructions

Upload the complete folder, not individual HTML files. The folder is self-contained and contains website pages plus PDFs in the `downloads` directory.

Fastest: Netlify Drop

  1. Open app.netlify.com/drop.
  2. Drag this complete `Fellowship-Bewerbung_FINAL` folder onto the page.
  3. Wait for the deploy URL.
  4. Paste that URL into the application/link field.

GitHub Pages

  1. Create a public repository.
  2. Upload the folder contents.
  3. Enable Pages from the main branch/root.
  4. Use the generated `github.io` URL.

Cloudflare Pages

  1. Create a Pages project.
  2. Use direct upload.
  3. Upload the complete folder.
  4. Use the deployment URL.
Do not upload the internal `portfolio_rebuild_control` folder, raw source folders, `.env` files, WhatsApp/OpenClaw tokens, private source paths, or old draft folders.
Prepared as an evidence-first portfolio package. Public files intentionally omit private source paths, credentials, private contact data, and raw local links.
